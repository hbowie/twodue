Software Versions

Version 1.4

  1. There is now a new Time Tab with information about the time that a particular item is expected to occur. This Tab has three fields: Start Time, Duration, and Alert Prior. There is also a default Start Time on the View Tab. Whatever Start Time you enter here (00:00 or midnight, by default) will be assigned to each new To Do Item within that particular data store.
  2. By setting a start time for an item and by setting alert prior to something other than N/A (both fields appearing on the new Time Tab), an alert will be triggerred at the specified time. Your system will beep and a dialog box will come up, reminding you of the item. You may dismiss the box with no change, or request that the alerting item be brought into view.

Version 1.3

  1. Fixed a bug in logic that checked your registration code. If the first digit after the last dash was a zero, then the registration check would fail.
  2. Can now set a recurrence to happen on the last day of week of a month (in addition to the first, second, third or fourth).
  3. Added a category field. Multiple category levels can be separated by a forward slash ('/') or a period ('.'). The new Tree Tab allows you to see your items organized by categories.

Version 1.2

  1. Added information about the current Java Run-time to the bottom of the About Tab.
  2. Added a Web Page field for each To Do item, along with a button to launch the Web page in your Web browser.
  3. Added access to Kagi Registration page from Help menu.
  4. Added access to Two Due User Guide from Help menu.
